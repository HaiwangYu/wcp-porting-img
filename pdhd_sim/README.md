# pdhd_sim — ProtoDUNE-HD track simulation

Self-contained driver for the wire-cell `wct-sim-check-track.jsonnet` flow on
ProtoDUNE-HD. Given a single track described in JSON, it simulates ionisation
+ drift + response, runs Noise Filtering (NF), and writes the NF-output frames
as `tar.bz2` files (one per anode, tag `raw<N>`). Output frames have the same
on-disk format as the masked frames produced by `woodpecker run-img`, so they
can be fed into the same downstream NF/SP/imaging pipeline for a direct
data-vs-simulation comparison.

Originally adapted from `/nfs/data/1/xning/wirecell-working/wcp-porting-img/pdhd/`.

## Layout

```
pdhd_sim/
├── wct-sim-check-track.jsonnet   # entry-point cfg (HD: pgrapher/experiment/pdhd/...)
├── generate_tracks.py            # builds tracks/*.json from wires geometry
├── run_sim_track.sh              # simulation driver
├── extract_track_waveform.py     # 1D peak-aligned mean waveform extractor
├── run_plot.sh                   # plotting driver (full frame + waveform)
├── tracks/                       # per-anode-plane track JSONs (12 files)
│   └── tracks-hd-anode<N>-<P>.json
├── work/                         # outputs (auto-populated)
│   └── anode<N>-<P>/
│       ├── protodunehd-sp-frames-sim-anode<N>.tar.bz2
│       ├── *.tar.bz2.png                       # U/V/W frame view
│       ├── *.tar.bz2.<plane>-waveform.png      # peak-aligned mean waveform
│       ├── *.tar.bz2.<plane>-waveform.npy      # waveform array (length 400)
│       └── wct.log
└── README.md
```

The cfg files imported by the jsonnet (`pgrapher/experiment/pdhd/simparams.jsonnet`,
`sim.jsonnet`, `nf.jsonnet`, `chndb-base.jsonnet`, etc.) live in
`/nfs/data/1/xqian/toolkit-dev/toolkit/cfg/` and are referenced via
`WIRECELL_PATH` set by the run script — nothing depends on
`/nfs/data/1/xning/...` at run time.

## What the jsonnet does

`wct-sim-check-track.jsonnet`:

1. Reads a `tracks_json` array (passed via `--tla-code`), picks the **longest**
   track, builds a single 0.1 mm-step deposition along it (`charge=-500`, MIP).
2. Drifts → bags depos → fans out to per-anode pipelines.
3. Each pipeline: `splusn` (sim + electronics + noise) → NF →
   `FrameFileSink` writing tag `raw<ident>`.
4. Uses the HD `lar` overrides (DL=6.2, DT=16.3, lifetime=50 ms,
   drift_speed=1.565 mm/μs) baked into the jsonnet itself, matching
   `pgrapher/experiment/pdhd/wct-sim-check.jsonnet`.

## Generating the track JSONs

`tracks/` ships pre-populated, but you can regenerate it from the wires
geometry:

```
./generate_tracks.py                              # writes 12 files into tracks/
./generate_tracks.py --drift-offset-mm 300        # tune drift offset
./generate_tracks.py --geom /path/to/wires.json.bz2 --outdir other_dir/
```

Each track is parallel to the wire plane (constant x), perpendicular to that
plane's wires, runs through the geometric centre of the plane bbox, and
extends edge-to-edge along the pitch direction. The default geometry is
`/nfs/data/1/xqian/toolkit-dev/wire-cell-data/protodunehd-wires-larsoft-v1.json.bz2`.
The default drift offset (500 mm from the wire plane toward the cathode)
keeps the deposition inside the field-response region — see the docstring
for the trade-off.

## Running

```
./run_sim_track.sh                  # all 4 anodes × 3 planes = 12 runs
./run_sim_track.sh -a 0             # all planes for anode 0
./run_sim_track.sh -a 0 -p W        # one combo
./run_sim_track.sh -g 7.8           # use low-gain (default 14 mV/fC)
```

Each `(anode N, plane P)` combo reads `tracks/tracks-hd-anode<N>-<P>.json` and
writes to `work/anode<N>-<P>/`.

## Output

`work/anode<N>-<P>/protodunehd-sp-frames-sim-anode<N>.tar.bz2` — NF-output
frames with tag `raw<N>`, no digitisation, no masks. `wct.log` has the
debug-level wire-cell log.

## Plotting

```
./run_plot.sh                       # plot every existing combo
./run_plot.sh -a 0 -p W             # one combo
./run_plot.sh --frames-only         # skip waveform extraction
./run_plot.sh --waveform-only       # skip full-frame PNG
```

Two artefacts per archive:

- `*.tar.bz2.png` — U/V/W full-frame view. Generated by `woodpecker plot-frames`
  (the `woodpecker` CLI on PATH, installed from
  `/nfs/data/1/xqian/toolkit-dev/Woodpecker/`).
- `*.tar.bz2.<plane>-waveform.{png,npy}` — peak-aligned mean waveform across
  channels with `|peak| > 5*RMS` on the target plane. Generated by the local
  `extract_track_waveform.py`, which reuses
  `_load_frames`/`_aligned_mean_waveform_full` from
  `Woodpecker/woodpecker/cli/cmd_compare_waveforms.py`. The script exists
  because xqian's woodpecker registers `compare-waveforms` but not the newer
  standalone `extract-track-waveform` subcommand.

The target plane is parsed from the parent dir name (`anode<N>-<P>`), so the
default just works after `run_sim_track.sh`. Override with `--plane U|V|W`.
